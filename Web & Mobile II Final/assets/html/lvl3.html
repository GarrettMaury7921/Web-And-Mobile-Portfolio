Security



Unix systems are one of the most widely used across the world. One of the most common operating systems for servers is the Linux OS of the Unix family. Because of this, hackers know how to hack into this specific OS better than others. Therefore, it is important to make sure that you take some precautions with your Linux device. There are some very common security vulnerabilities that can be fixed with a little bit of Linux knowledge. First is to close any unnecessary ports. These ports can be taken advantage of if they are managed poorly, so, it is best to just shut down any ports if you know that you will never use them.

To get an active list of your ports use the command: netstat -tulp

You should also scan your machine to learn what ports are in listening mode. These ports are extra vulnerable as they are waiting for a connection with the outside world.

To scan for listening ports use the command: nmap -sTU “your host machine”

Like ports, it is important to manage the services that your have running on your Linux device. These services can be used improperly and can even give an attacker access to your system.

To list the services running use the command: chkconfig -list | awk

To stop a service permanently use the command: chkconfig “service name” off

It is also important to change your password, especially the password for the root account. If you have lots of accounts on your machine it is essential that all these passwords are not default and are somewhat complex.

To change your password, use the command: passwd

File System Security



Every file on a Unix system has access rights associated with it. Access focuses on three main rights – read, write and execute. Each of these rights plays an important role in creating a safe and well-organized system. The read right only allows a user to look at the data in a file/folder. The write right only allows a user to write new data to a file/folder. The execute right only allows a user to run programs in a file/folder. These three rights can be used in combo to provide the level of access that the administrator decides is appropriate.

You can view the rights associated with the files in your current directory by using the command: ls -l

The information returned with this command is the rights for three groups. The owner of the system, the group that created the file and everyone else. The 9 characters are the 3 rights held by these three groups. These settings can be changed upon your desire to let other people access the information on your system.
